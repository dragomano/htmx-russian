Атрибут `hx-delete` заставит элемент отправить `DELETE`-запрос по указанному URL и заменить HTML в DOM, используя стратегию замены:

```html wrap
<button hx-delete="/account" hx-target="body">
  Удалить учётку
</button>
```

Этот пример заставит `button` отправить `DELETE`-запрос на `/account` и заменить возвращаемый HTML в `innerHTML` элемента `body`.

## Примечания

- `hx-get` не наследуется.
- Вы можете контролировать цель замены с помощью атрибута [hx-target](../hx-target).
- Вы можете контролировать стратегию замены, используя атрибут [hx-swap](../hx-swap).
- Вы можете контролировать, какое событие вызывает запрос, с помощью атрибута [hx-trigger](../hx-trigger).
- Вы можете контролировать данные, отправляемые с запросом, различными способами, описанными здесь: [Параметры](/ajax/parameters).
- Чтобы удалить элемент после успешного `DELETE`, верните код состояния `200` с пустым телом; если сервер отвечает кодом `204`, замена не происходит, как описано здесь: [Запросы и ответы](/requests).
